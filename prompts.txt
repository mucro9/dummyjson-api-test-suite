## [2026-06-28T16:40:00Z]
PROMPT:
[Ran the collection in the Postman app Collection Runner; 73 passed / 2 failed. Shared two
screenshots: the 2 failures are both on "Get current user — no token (401)" — expected 401 but
got 200. Asked Claude to analyze.]

RESPONSE SUMMARY:
Diagnosed the 2 failures as the documented HttpOnly cookie-jar trap surfacing differently in the
Postman app than in Newman. Probed the live API with curl to confirm the tests are correct: no
token/cookie -> 401, but a request carrying only the login cookie (no auth header) -> 200, so
the leftover cookie silently authenticates. Newman runs 75/75 because its cookie jar is cleared
by the pre-request script; the Postman app blocks programmatic cookie-jar access for
non-allowlisted domains, so the clear silently no-ops and the stale cookie leaks in. Fix:
allowlist dummyjson.com for programmatic cookie access in the app (guiding the user), then
re-run for a clean 75/75; documenting the app-only setup note while keeping Newman/CI zero-config.
